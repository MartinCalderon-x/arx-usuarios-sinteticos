## La Historia: Por qué existe esta plataforma

### El Problema

Imagina que eres diseñador UX en una empresa que lanza una nueva app de banca móvil. Antes de salir a producción, necesitas responder preguntas críticas:

- *"¿Mi abuela de 70 años podrá encontrar cómo pagar sus servicios?"*
- *"¿Un millennial tech-savvy se frustrará con tantos pasos?"*
- *"¿Dónde mira primero el usuario cuando abre la pantalla?"*

Tradicionalmente, responder estas preguntas requiere:
- **Estudios de eye-tracking**: $5,000-$15,000 USD por estudio
- **Pruebas de usabilidad**: 4-6 semanas de reclutamiento y ejecución
- **Entrevistas con usuarios**: Coordinación compleja, sesiones de 1-2 horas

Y después de todo ese tiempo y dinero, solo tienes datos de 8-12 participantes.

### La Solución

**Usuarios Sintéticos** combina dos tecnologías de IA para democratizar la investigación UX:

1. **Modelos de Atención Visual** - Predicen dónde mirará el usuario en los primeros segundos, con 87-92% de correlación con eye-tracking real.

2. **Arquetipos de IA** - Simulan cómo diferentes tipos de usuarios navegarían tu interfaz, identificando fricciones antes de que lleguen a producción.

El resultado: **En 5 minutos obtienes insights que antes tomaban semanas**.

## ¿Qué puedes hacer con Usuarios Sintéticos?

### 1. Predecir Atención Visual (Heatmaps)

Sube cualquier diseño y obtén un mapa de calor que muestra dónde se concentrará la atención visual del usuario.

**¿Cómo funciona?**
La plataforma combina tres modelos de IA:
- **DeepGaze III** - Red neuronal entrenada con millones de datos de eye-tracking
- **Itti-Koch** - Modelo neurocientífico de atención biológica
- **Gemini Vision** - Comprensión semántica de elementos importantes

**¿Qué obtienes?**
- **Heatmap**: Colores cálidos = alta atención, fríos = baja atención
- **Focus Map**: Overlay sobre tu diseño original
- **Clarity Score**: Puntuación 0-100 de qué tan claro es tu diseño
- **Áreas de Interés**: Lista ordenada de qué elementos captarán atención primero
- **Insights**: Observaciones automáticas sobre tu diseño

**Precisión:**
- Primeros 0-3 segundos: 92% correlación con eye-tracking real
- 3-5 segundos: 87% correlación
- Ideal para landing pages, anuncios, packaging, dashboards

### 2. Crear Arquetipos de Usuario

Los arquetipos son representaciones de diferentes tipos de usuarios con características específicas que afectan cómo interactúan con interfaces.

**Características de un arquetipo:**
- **Datos demográficos**: Edad, género, ocupación
- **Nivel digital**: Bajo, medio o alto (afecta directamente las simulaciones)
- **Frustraciones**: Qué les molesta de las interfaces
- **Objetivos**: Qué buscan lograr
- **Comportamiento**: Cómo suelen navegar

**15 Templates Predefinidos:**

| Categoría | Arquetipos |
|-----------|------------|
| **Consumidor Digital** | Early Adopter, Usuario Promedio, Escéptico Digital, Senior Tradicional, Comprador Impulsivo |
| **Manufactura** | Operario de Línea, Supervisor de Planta, Ingeniero de Procesos |
| **Retail** | Comprador Impulsivo, Comparador de Precios, Cliente Fiel |
| **Finanzas** | Inversionista Novato, Usuario Fintech, Cliente Banca Tradicional |
| **Salud** | Paciente Crónico, Cuidador Familiar |

**Crear desde Datos Reales:**
Si tienes transcripciones de entrevistas, encuestas o reportes de investigación, la plataforma puede extraer automáticamente las características para crear arquetipos basados en usuarios reales.

### 3. Conversar con Usuarios Sintéticos

Puedes tener conversaciones en tiempo real con cualquier arquetipo para obtener feedback sobre diseños.

**Ejemplo de conversación:**

> **Tú**: "¿Qué opinas de esta pantalla de checkout?"
>
> **Senior Tradicional (María, 68 años)**: "Me siento un poco perdida. Hay muchas opciones y no sé cuál elegir primero. El botón de 'Pagar' es muy pequeño y está hasta abajo. Preferiría que estuviera más visible. También me gustaría ver el total antes de hacer click en cualquier cosa."

**Cada respuesta incluye:**
- Feedback desde la perspectiva del arquetipo
- Fricciones identificadas
- Estado emocional (frustración, confianza, confusión)

---

### 4. Flujos Multi-Pantalla

Un flujo es una secuencia de pantallas que representan un journey del usuario, como "Proceso de compra" o "Registro de cuenta".

**Para cada pantalla del flujo obtienes:**
- Heatmap de atención visual
- Clarity Score
- Áreas de interés
- Elementos interactivos detectados

**Uso típico:**
1. Creas un flujo llamado "Checkout e-commerce"
2. Subes 5 pantallas: Carrito → Datos → Envío → Pago → Confirmación
3. La plataforma analiza cada pantalla automáticamente
4. Puedes ver dónde se pierde atención en cada paso

---

### 5. Usability Testing con Simulación (NUEVO)

Esta es la funcionalidad más poderosa: simular pruebas de usabilidad completas sin usuarios reales.

**¿Cómo funciona?**

1. **Defines una Misión**: "Encuentra el botón de contacto" o "Completa una compra"

2. **Detectas Elementos**: La IA identifica todos los elementos clickeables en cada pantalla (botones, links, inputs, menús, iconos)

3. **Seleccionas Arquetipos**: Eliges qué tipos de usuarios quieres simular

4. **Ejecutas Simulación**: Cada arquetipo "navega" tu flujo intentando completar la misión

5. **Analizas Resultados**:
   - **Success Rate**: % de usuarios que completaron la tarea
   - **Misclicks**: Clicks en elementos incorrectos
   - **Path Tomado**: Secuencia exacta de clicks
   - **Fricciones**: Puntos donde los usuarios se confundieron
   - **Feedback**: Comentarios desde la perspectiva de cada arquetipo

**Diferenciación por Nivel Digital:**

| Nivel | Success Rate Esperado | Misclicks | Comportamiento |
|-------|----------------------|-----------|----------------|
| **Bajo** | 40-60% | 2-4 | Se confunde con iconos, evita opciones complejas, busca elementos grandes |
| **Medio** | 70-85% | 1-2 | Navegación normal, prefiere textos claros |
| **Alto** | 90-98% | 0-1 | Eficiente, identifica CTAs rápido, tolera complejidad |

**Reproductor de Simulación:**
Puedes ver paso a paso cómo cada arquetipo navegó tu flujo:
- Qué elemento clickeó
- Por qué lo eligió (razonamiento)
- Si fue un misclick
- Su estado emocional en cada paso

## Casos de Uso

### Caso 1: Optimizar Landing Page

**Situación**: Tienes una landing page para un producto SaaS y la conversión es baja.

**Con Usuarios Sintéticos:**
1. Subes el diseño actual
2. Obtienes el heatmap → Descubres que el CTA principal está en una zona fría
3. Ves que el "Clarity Score" es 45 → Hay demasiados elementos compitiendo
4. Los insights sugieren: "El botón de acción está por debajo del fold y compite con elementos secundarios"

**Resultado**: Mueves el CTA arriba y eliminas distractores. Conversión sube 23%.

---

### Caso 2: Validar Flujo de Onboarding

**Situación**: Diseñaste un nuevo flujo de registro en 4 pasos. Antes de implementar, quieres validar.

**Con Usuarios Sintéticos:**
1. Creas un flujo con las 4 pantallas del onboarding
2. Defines misión: "Completa el registro"
3. Simulas con 3 arquetipos: Early Adopter, Usuario Promedio, Senior Tradicional
4. Resultados:
   - Early Adopter: 95% success, 0 misclicks
   - Usuario Promedio: 80% success, 1 misclick en paso 3
   - Senior Tradicional: 45% success, 3 misclicks, abandono en paso 2

**Descubrimiento**: El paso 2 tiene un selector de fecha que confunde a usuarios de bajo nivel digital. El campo no tiene label visible y usa un formato poco familiar.

**Resultado**: Agregas labels claros y cambias el selector. Success rate del Senior sube a 78%.

---

### Caso 3: Comparar Diseños A/B

**Situación**: Tienes dos versiones de un checkout y no sabes cuál es mejor.

**Con Usuarios Sintéticos:**
1. Analizas ambos diseños con heatmaps
2. Diseño A: Clarity Score 72, CTA en zona caliente
3. Diseño B: Clarity Score 58, atención dispersa

**Resultado**: Implementas Diseño A con confianza antes de hacer un A/B test costoso con usuarios reales.

---

### Caso 4: Preparar Presentación para Stakeholders

**Situación**: Necesitas justificar cambios de diseño a directivos no técnicos.

**Con Usuarios Sintéticos:**
1. Generas heatmaps del diseño actual vs propuesto
2. Muestras simulaciones: "Miren cómo María de 68 años no puede encontrar el botón"
3. Presentas métricas: "Success rate actual 45%, propuesto 82%"

**Resultado**: Aprobación inmediata basada en evidencia visual y métricas claras.

---

### Caso 5: Crear Arquetipos desde Investigación Real

**Situación**: Hiciste 12 entrevistas con usuarios y tienes transcripciones de 2 horas cada una.

**Con Usuarios Sintéticos:**
1. Subes los documentos (PDF, DOCX, TXT)
2. La IA extrae patrones: frustraciones comunes, objetivos, comportamientos
3. Genera un arquetipo representativo con citas textuales de las entrevistas
4. Nivel de confianza: 0.85 (alta)

**Resultado**: En 2 minutos tienes un arquetipo sintético basado en 24 horas de entrevistas.

## Guía de Funcionalidades

### Dashboard Principal

Al iniciar sesión ves un resumen de:
- Arquetipos creados
- Análisis realizados
- Flujos activos
- Reportes generados

---

### Módulo: Arquetipos

**Crear Arquetipo Manual:**
1. Click en "Nuevo Arquetipo"
2. Selecciona un template o empieza desde cero
3. Completa: nombre, edad, ocupación, nivel digital
4. Agrega frustraciones y objetivos (mínimo 2 de cada uno)
5. Guarda

**Crear desde Datos:**
1. Click en "Crear desde datos"
2. Sube hasta 5 archivos (PDF, DOCX, TXT, CSV)
3. Máximo 10MB por archivo
4. La IA procesa y extrae características
5. Revisa y ajusta el arquetipo sugerido
6. Guarda

**Chatear con Arquetipo:**
1. Abre un arquetipo
2. En el panel de chat, escribe tu pregunta
3. Opcionalmente adjunta una imagen de diseño
4. Recibe feedback desde la perspectiva del arquetipo

---

### Módulo: Análisis Visual

**Analizar Imagen:**
1. Click en "Nuevo Análisis"
2. Sube una imagen (PNG, JPG, WebP)
3. Espera el procesamiento (10-30 segundos)
4. Explora resultados:
   - Tab "Original": Tu imagen
   - Tab "Heatmap": Mapa de calor
   - Tab "Overlay": Heatmap sobre tu imagen
5. Panel derecho: Clarity Score, Áreas de Interés, Insights

**Comparar Modelos:**
1. Ve a "Comparar Modelos"
2. Sube una imagen
3. Compara resultados de DeepGaze III vs Modelo Híbrido
4. Ve métricas de correlación y diferencias

---

### Módulo: Flujos

**Crear Flujo:**
1. Click en "Nuevo Flujo"
2. Nombre y descripción
3. Guarda

**Agregar Pantallas:**
1. Abre el flujo
2. Click en "Agregar Pantalla"
3. Arrastra imagen o selecciona archivo
4. La pantalla se analiza automáticamente
5. Repite para cada pantalla del journey

**Ver Análisis de Pantalla:**
1. Click en una pantalla del timeline
2. Cambia entre vistas: Original, Heatmap, Overlay, Elementos
3. Panel derecho muestra Clarity Score, Áreas, Insights

**Detectar Elementos:**
1. Selecciona una pantalla
2. Click en "Detectar" (icono de escáner)
3. La IA identifica todos los elementos clickeables
4. Ve el resultado en vista "Elementos"
5. Los elementos se muestran con bounding boxes de colores por tipo

---

### Módulo: Misiones (Usability Testing)

**Crear Misión:**
1. En un flujo, ve al tab "Misiones"
2. Click en "Nueva Misión"
3. Define:
   - Nombre: "Completar compra"
   - Instrucciones: "Encuentra y haz click en el botón de pagar"
   - Pantalla inicio: Primera pantalla del flujo
   - Pantalla objetivo: Donde está el elemento a encontrar
   - Elemento objetivo: "Pagar" (botón)
   - Máximo pasos: 10
4. Guarda

**Ejecutar Simulación:**
1. En la misión, click en "Simular"
2. Selecciona uno o más arquetipos
3. Click en "Ejecutar"
4. Espera el procesamiento (10-60 segundos por arquetipo)
5. Ve resultados automáticamente

**Analizar Resultados:**
1. Ve la lista de simulaciones ejecutadas
2. Click en una simulación para ver el detalle
3. Usa el reproductor para ver paso a paso
4. Revisa fricciones detectadas
5. Lee el feedback del arquetipo

**Dashboard de Usability:**
1. Click en "Dashboard" en el header del flujo
2. Ve métricas agregadas de todas las misiones
3. Compara success rate por nivel digital
4. Lee recomendaciones automáticas

---

### Módulo: Reportes

**Generar Reporte:**
1. Ve a "Reportes"
2. Click en "Generar Reporte"
3. Selecciona:
   - Título
   - Formato (PDF o PPTX)
   - Arquetipos a incluir
   - Análisis a incluir
   - Sesiones de chat a incluir
4. Genera y descarga

## Métricas Clave

### Clarity Score (0-100)

| Rango | Interpretación | Acción |
|-------|----------------|--------|
| 80-100 | Excelente | Diseño claro, jerarquía visual definida |
| 60-79 | Bueno | Funcional, algunas mejoras posibles |
| 40-59 | Regular | Atención dispersa, revisar jerarquía |
| 0-39 | Pobre | Rediseñar, demasiados elementos compitiendo |

### Success Rate (Simulaciones)

| Rango | Interpretación | Acción |
|-------|----------------|--------|
| 90-100% | Excelente | Flujo intuitivo para todos los niveles |
| 70-89% | Bueno | Funcional, optimizar para niveles bajos |
| 40-69% | Regular | Fricciones significativas, revisar UX |
| 0-39% | Crítico | Rediseñar el flujo completo |

### Misclicks Promedio

| Rango | Interpretación | Acción |
|-------|----------------|--------|
| 0-1 | Óptimo | Elementos claros y bien posicionados |
| 1-2 | Aceptable | Algunos elementos confusos |
| 2-4 | Problemático | Revisar tamaños y espaciado |
| 4+ | Crítico | Elementos demasiado pequeños o confusos |

## Mejores Prácticas

### Para Análisis Visual
- Usa imágenes de alta resolución (mínimo 1280x720)
- Analiza diseños en su contexto real (con contenido, no lorem ipsum)
- Compara versiones antes y después de cambios

### Para Arquetipos
- Incluye al menos 3 frustraciones y 3 objetivos
- Varía los niveles digitales para cubrir todo tu público
- Basa arquetipos en datos reales cuando sea posible

### Para Simulaciones
- Define misiones específicas y medibles
- Ejecuta con arquetipos de diferentes niveles digitales
- Itera: detecta fricciones → ajusta diseño → vuelve a simular

### Para Flujos
- Ordena pantallas en secuencia lógica del journey
- Detecta elementos en TODAS las pantallas antes de simular
- Usa nombres descriptivos para pantallas

## Limitaciones

### Atención Visual
- Precisión disminuye después de 5 segundos de viewing
- No predice comportamiento "task-driven" (cuando el usuario busca algo específico)
- Mejor para "free-viewing" (exploración inicial)

### Simulaciones
- Los arquetipos son aproximaciones basadas en características
- No reemplazan pruebas con usuarios reales, las complementan
- Mejor como validación temprana, antes de desarrollo costoso

### Detección de Elementos
- Precisión ~85% en interfaces estándar
- Puede fallar con diseños muy creativos o no convencionales
- Siempre verifica y corrige manualmente si es necesario

## Flujo de Trabajo Recomendado

```
1. DISEÑO INICIAL
   └─→ Análisis de heatmap
   └─→ Clarity Score > 60?
       ├─ NO → Iterar diseño
       └─ SI → Continuar

2. CREACIÓN DE FLUJO
   └─→ Subir todas las pantallas
   └─→ Detectar elementos en cada una
   └─→ Verificar detección

3. VALIDACIÓN CON ARQUETIPOS
   └─→ Crear misiones clave
   └─→ Simular con 3+ arquetipos (bajo, medio, alto)
   └─→ Success rate > 70% en todos?
       ├─ NO → Identificar fricciones, iterar
       └─ SI → Listo para desarrollo

4. DOCUMENTACIÓN
   └─→ Generar reporte con hallazgos
   └─→ Compartir con equipo/stakeholders
```
